# Contributing

## Adding a project

1. Create a directory under `projects/<your-project-name>/`
2. Add a `README.md` with:
   - One-line description
   - Who's on the team
   - How to run it locally
   - What's working / what's not
3. Open a PR — or push directly to `main` if you have access. Don't block on review.

## Folder layout

```
projects/
  your-project-name/
    README.md
    <whatever you need>
```

Languages, frameworks, and structure inside your project directory are up to you. Use what gets you shipping fastest.

## Ground rules

- **Ship something.** A rough working demo beats a polished plan.
- **Show your work.** Commit early, commit often. Half-finished is fine.
- **No secrets in commits.** Use `.env.local` or similar, gitignored.
- **Be kind in PR reviews.** This is a hackathon, not prod.

## Questions

Open an issue or ping the team channel.
